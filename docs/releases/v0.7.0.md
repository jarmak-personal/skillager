# skillager v0.7.0

Packages:
- `skillager` `0.7.0`
- `skillager-linter` is unchanged at `0.1.2`; no linter API, CLI, or validation behavior changed in this release.

## Highlights

`0.7.0` is the CLI-pruning release. It makes Skillager's public surface smaller for humans and more predictable for agents, with breaking command removals accepted intentionally.

- `materialize` is now `expose`.
- Approval now lives under `review`: `review approve`, `review pin`, `review block`, and `review unblock`.
- Human-facing setup and diagnostics center on `setup`, `doctor`, `review`, `expose`, `tag`, and `collection`.
- Agent-facing protocol commands are `working`, `list`, `search`, `show`, and `activate`.
- Removed pre-pruning command names now fail with normal argparse invalid-choice errors instead of long-lived compatibility shims.

## Exposure

- `expose` supports native, stub, and router modes.
- Explicit router exposure can route a named set of skills without requiring a tag.
- Router activation now validates against router sidecar membership, and stale or unavailable router members degrade per member instead of failing the whole router.
- `expose` resolves the target agent from explicit flags, the detected agent, or saved setup state when unambiguous.
- `expose --all-reviewed` now requires an explicit mode and rejects `--mode native` to avoid accidental context bloat.

## Setup And Review

- `setup --collection <name> --bulk-approve` is the short path for company-maintained collections, with `--yolo` kept as an alias.
- `--scope` defaults to `project`.
- `--collection <name>` implies collection source selection for review/setup flows.
- Agents key off availability and exposure, not owner approval vocabulary.

## Removed Or Folded Commands

These names are no longer public CLI commands: `trust`, `block`, `handoff`, `bootstrap`, `status`, `index`, `scan`, `project`, `new`, `manifest`, runtime `lint`, and `state`.

Use:

- `review approve/pin/block/unblock` for owner review decisions.
- `doctor` for human diagnostics and first-party repairs.
- `working --json` for agent readiness.
- `expose` for native, stub, and router exposure.
- `tag` and `collection` for curation and collection administration.
- `skillager-lint` from `skillager-linter` for standalone author linting.
